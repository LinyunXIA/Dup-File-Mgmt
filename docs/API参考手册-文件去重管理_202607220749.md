## 0. 全局约定

| 项 | 说明 |
|---|---|
| Base URL | `http://localhost:5001`（生产请改为本机 IP） |
| Content-Type | `application/json`（除 `/api/import` 为 `multipart/form-data`） |
| 时间字段 | 字符串格式（SQLite 直接 str 化），可为 `null` |
| 状态字段 | task 状态：`pending` / `executed` / `cancelled` |
| 公共返回字段 | 多数查询接口都带 `readonly` / `library_locked` / `pending_task_exists` / `pending_task_count` |
| 错误响应 | `{"error": "<中文消息>"}` + HTTP 状态码；锁定类错误另带 `"code": 403` |
| ID 类型 | 所有 ID 均为 INTEGER（SQLite 自增主键） |
| 路径格式 | 库内只存服务器原始路径（如 `/volume1/...`），不做调整 |
| 协议限制 | ▶️ 播放仅允许 `smb://` 协议（防 `file://` / 任意 shell 注入） |

### 错误响应统一格式

```json
// 通用错误
{ "error": "缺少 task_id" }

// 锁定类错误（额外带 code）
{ "error": "有 pending task 未处理，请先到 Task 页面处理", "code": 403 }
```

## 2. 导入

### 2.1 `POST /api/import`

上传并导入 `md5_list.txt`。

**请求**：`multipart/form-data`

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `file` | form | File | ✅ | md5_list.txt（单文件上限 500MB，由 Flask `MAX_CONTENT_LENGTH` 强制） |
| `mode` | form | string | ❌ 默认 `first` | `first`=全量导入；`incremental`=增量（**当前未实现过滤逻辑**，等同 first，见产品文档 12.1） |

**响应 200**：
```json
{
  "ok": true,
  "inserted": 12345,         // 新增行数
  "updated": 678,            // 路径命中 UNIQUE 但元数据被更新
  "skipped_ds_store": 23,    // 跳过的 .DS_Store（mac 系统元数据）
  "errors": 0,               // 解析失败行数
  "total_lines": 13046,      // 文件总行数
  "elapsed_ms": 1234,        // 耗时（毫秒）
  "duplicate_ids": [1,5,12]  // inserted>0 时返回：本次导入后处于 MD5 重复组的 file.id 列表
}
```

**关键行为**：
- 按 `BATCH_SIZE = 1000` 行事务批量 INSERT（`ON CONFLICT(file_path) DO UPDATE`）
- 路径用 `split('  ', 1)` 切 MD5 和路径（兼容空格路径）
- 自动跳过 `.DS_Store`（路径含 `/.DS_Store` 或以 `.DS_Store` 结尾）
- 导入完成后扫一次重复组（GROUP BY md5 HAVING COUNT(*) > 1），返回相关 id

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "请选择文件"}`（无 file 字段 或 file.filename 为空）|
| 413 | Flask `MAX_CONTENT_LENGTH` 触发（> 500MB） |

## 3. 查重浏览

### 3.1 `GET /api/duplicates`

列出所有 MD5 重复组（出现次数 > 1）。

**请求**：无参数

**响应 200**：
```json
{
  "groups": [
    {
      "md5": "178e097c50395b96bdf286f1d8f88a44",
      "md5_short": "178e097c5039",
      "count": 2,
      "total_size": 1048576,
      "files": [
        {
          "id": 1,
          "file_path": "/volume1/GV/a.mp4",
          "file_name": "a.mp4",
          "parent_dir": "/volume1/GV",
          "md5": "178e097c50395b96bdf286f1d8f88a44",
          "file_size": 524288,
          "imported_at": "2026-07-22 07:30:00",
          "is_locked": true
        }
      ]
    }
  ],
  "readonly": false,
  "library_locked": true,
  "locked_group_count": 343,
  "pending_task_exists": false,
  "pending_task_count": 0
}
```

**排序**：`ORDER BY md5, file_name`

**字段说明**：
- `is_locked` = 该文件的 MD5 处于锁定状态（用于前端单文件行 disabled 判定）
- `locked_group_count` = 整库处于锁定的 MD5 组数（库级锁定判定）

---

### 3.2 `GET /api/same-name`

列出"文件名相同但 MD5 不同"的组（Phase 2）。

**请求**：无参数

**响应 200**：
```json
{
  "groups": [
    {
      "file_name": "1.mp4",
      "count": 3,
      "has_ignored": false,
      "files": [
        {
          "id": 10,
          "file_path": "/dir1/1.mp4",
          "parent_dir": "/dir1",
          "md5": "abc...",
          "md5_short": "abc...",
          "file_size": 100,
          "ignored_at": null
        }
      ]
    }
  ],
  "readonly": false,
  "pending_task_exists": false,
  "pending_task_count": 0
}
```

**过滤规则**（已实现）：
1. `file_name` 出现次数 ≥ 2
2. 组内 `ignored_at` **全为非 NULL** → **跳过整组**
3. 组内**至少 1 条** `ignored_at` 为 NULL → **整组列出**（含已忽略行，`has_ignored=true`）

**注意**：与 `/api/duplicates` 不同，**不返回** `library_locked` / `locked_group_count`（同名组与 MD5 重复无直接关联）。

## 4. 文件浏览

### 4.1 `GET /api/tree`

返回目录树（按 `parent_dir` 聚合）。

**请求 Query**：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `q` | string | ❌ | 模糊匹配 `file_name` 或 `file_path`（LIKE %q%） |

**响应 200**：
```json
{
  "directories": [
    {"path": "/volume1/GV", "count": 931},
    {"path": "/volume1/GV/20240825", "count": 2755}
  ],
  "total_files": 12872,
  "readonly": false,
  "library_locked": true,
  "locked_group_count": 343,
  "locked_md5_set": ["178e097c...", "ab12cd34..."],
  "pending_task_exists": false,
  "pending_task_count": 0
}
```

**搜索行为**：传入 `q` 时只返回 `file_name LIKE %q%` OR `file_path LIKE %q%` 命中的目录

---

### 4.2 `GET /api/files`

列出文件，支持过滤、搜索、排序。

**请求 Query**：

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `parent_dir` | string | `""`（=全部） | 精确匹配目录 |
| `q` | string | `""` | 模糊匹配 `file_name` / `file_path` |
| `sort_by` | enum | `parent_dir` | 允许值：`parent_dir` / `file_name` / `file_size` / `md5`；其他值被忽略，回退默认 |
| `sort_order` | enum | `asc` | `asc` / `desc`（大小写不敏感） |

**响应 200**：
```json
{
  "files": [
    {
      "id": 1,
      "file_path": "/volume1/GV/a.mp4",
      "file_name": "a.mp4",
      "parent_dir": "/volume1/GV",
      "md5": "178e097c50395b96bdf286f1d8f88a44",
      "md5_short": "178e097c5039",
      "file_size": 524288,
      "is_locked": true
    }
  ],
  "readonly": false,
  "library_locked": true,
  "locked_md5_set": ["178e097c..."],
  "pending_task_exists": false,
  "pending_task_count": 0
}
```

**SQL 模板**：
```sql
SELECT id, file_path, file_name, parent_dir, md5, file_size
FROM files
WHERE [parent_dir = ?] [AND (file_name LIKE ? OR file_path LIKE ?)]
ORDER BY {sort_by} {sort_order}
```

## 5. 删除 rm 操作

### 5.1 `POST /api/generate-rm`

由文件 ID 生成 rm 命令，同时创建 pending task。

**请求 Body**：
```json
{ "ids": [1, 5, 12] }
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `ids` | int[] | ✅ | `files.id` 列表，至少 1 个 |

**响应 200**：
```json
{
  "task_id": 7,
  "command": "rm '/volume1/GV/a.mp4'\nrm '/volume1/GV/b.mp4'",
  "count": 2,
  "paths": ["/volume1/GV/a.mp4", "/volume1/GV/b.mp4"]
}
```

**关键逻辑**：
- `command` 用 `\n` 分隔，**1 文件 1 行**
- 路径用 `shlex.quote` 转义（防空格 / 特殊字符）
- 立即创建 `pending` task，写入 `task_files` 关联

**校验**：
- 同 MD5 选中数 < 该 MD5 总数（**至少保留 1 个**）

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "请选择至少一个文件"}`（空 ids）|
| 404 | `{"error": "文件不存在"}` |
| 404 | `{"error": "部分文件不存在"}` |
| 400 | `{"error": "MD5 <12位> 必须至少保留 1 个文件，不能全选"}` |
| 403 | `{"error": "有 pending task 未处理，请先到 Task 页面处理", "code": 403}`（task 全局锁） |

---

### 5.2 `POST /api/confirm-rm`

确认 rm 已执行 → 写审计日志 + 删除记录。

**请求 Body**：
```json
{ "task_id": 7 }      // 也支持 { "id": 7 }（兼容）
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `task_id` | int | ✅ | 二选一（`task_id` 优先） |
| `id` | int | ✅ | 同上（兼容旧版） |

**响应 200**：
```json
{
  "deleted": 2,
  "readonly": false,
  "library_locked": true,
  "locked_group_count": 343
}
```

**副作用**（按顺序）：
1. 每个文件写一条 `audit_log`（action=`deleted`）
2. `DELETE FROM files WHERE id IN (...)`
3. `tasks.status` → `executed`，`executed_at` = NOW

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "缺少 task_id"}` |
| 403 | `{"error": "系统当前处于只读模式，请修改 config.json 中 readonly 为 false", "code": 403}` |
| 404 | `{"error": "task 不存在"}` |
| 400 | `{"error": "task 不是 pending 状态"}` |

**注意**：不再二次校验锁定（用户在弹窗内已看到锁定状态 + 主动点"已执行"才生效）。

## 6. 移动 mv 操作

### 6.1 `POST /api/generate-mv`

由文件 ID + 目标目录生成 mv 命令，同时创建 pending task。

**请求 Body**：
```json
{ "ids": [1, 5, 12], "target_dir": "/volume1/GV/dst" }
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `ids` | int[] | ✅ | 至少 1 个 |
| `target_dir` | string | ✅ | 目标目录（trim 后非空） |

**响应 200**：
```json
{
  "task_id": 8,
  "command": "mv -i '/volume1/GV/a.mp4' '/volume1/GV/dst/a.mp4' && \\\nmv -i '/volume1/GV/b.mp4' '/volume1/GV/dst/b.mp4'",
  "count": 2,
  "new_paths": ["/volume1/GV/dst/a.mp4", "/volume1/GV/dst/b.mp4"]
}
```

**关键逻辑**：
- `mv -i` 启用交互确认（防覆盖同名文件）
- 多条命令用 ` && \` 串联
- 目标文件名 = `os.path.join(target_dir, file_name)`（保留原文件名）
- 立即创建 pending task，关联 `target_dir` 字段

**校验**：
- 目标目录在库中存在（`SELECT COUNT(*) FROM files WHERE parent_dir = ?`）
- 任一 id 的 MD5 不在 `locked_md5_set` 中（库级锁定拦截）

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "请选择至少一个文件"}` |
| 400 | `{"error": "请选择目标目录"}` |
| 404 | `{"error": "文件不存在"}` |
| 400 | `{"error": "目标目录在库中不存在"}` |
| 403 | `{"error": "包含处于 md5 重复锁定的文件"}` |
| 403 | task 全局锁（消息同 generate-rm）|

---

### 6.2 `POST /api/confirm-mv`

确认 mv 已执行 → 写审计日志 + 更新路径。

**请求 Body**：
```json
{ "task_id": 8 }
```

**响应 200**：
```json
{
  "moved": 2,
  "readonly": false,
  "library_locked": true,
  "locked_group_count": 343
}
```

**副作用**（按顺序）：
1. 每个文件写一条 `audit_log`（action=`moved`，含 `new_path`）
2. `UPDATE files SET file_path=?, parent_dir=? WHERE id=?`
3. `tasks.status` → `executed`

**错误**：同 `confirm-rm`

## 7. 同名组忽略

### 7.1 `POST /api/ignore-same-name`

标记同名组为已忽略（强制全选）。

**请求 Body**：
```json
{ "file_ids": [10, 11, 12] }
```

**响应 200**：
```json
{ "ok": true, "ignored": 3 }
```

**副作用**：`UPDATE files SET ignored_at = CURRENT_TIMESTAMP WHERE id IN (...)`

**校验逻辑**（按 `file_name` 分组逐组校验）：
1. 所有选中 id 必须存在
2. 每个 `file_name` 仍 ≥ 2 条记录（确保是同名组）
3. **强制全选**：组内 `ignored_at IS NULL` 的所有记录必须全部选中
4. readonly 模式拒绝

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "请选择文件"}`（空 file_ids）|
| 403 | readonly 模式 |
| 400 | `{"error": "文件 <name> 不是同名组"}`（该 file_name 唯一）|
| 400 | `{"error": "有相同文件没有确认，请全部选中后再次点击忽略"}`（未全选）|

## 8. ▶️ 播放

### 8.1 `POST /api/play`

通过 SMB 协议在 macOS 触发默认播放器。

**请求 Body**：
```json
{ "file_id": 5 }
```

**响应 200**：
```json
{ "ok": true, "smb_url": "smb://192.168.88.212/GV/a.mp4" }
```

**关键逻辑**：
- 取 `file_path`（如 `/volume1/GV/a.mp4`）
- **剥第一级** → `/GV/a.mp4`
- 拼 `smb://<smb_ip>/GV/a.mp4`
- `subprocess.Popen(['open', smb_url])` 触发 macOS 默认应用

**安全**：
- 协议白名单：只允许 `smb://`（防 `file://` / 任意 shell 注入）
- 路径段校验：split 后必须 ≥ 2 段

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "缺少 file_id"}` |
| 503 | `{"error": "SMB 未配置，请先设置 config.json 中的 smb_server_ip"}`（ip 为 `0.0.0.0` 或缺失）|
| 404 | `{"error": "文件不存在"}` |
| 400 | `{"error": "路径格式不正确"}`（split 后 < 2 段）|
| 400 | `{"error": "仅支持 smb:// 协议"}`（理论不会触发，安全冗余）|
| 500 | `{"error": "打开播放器失败: <reason>"}`（subprocess 异常）|

**注意**：▶️ 播放**不受任何锁定影响**（只读，不改数据）。

### 9.2 `GET /api/task-detail`

获取单个 task 的命令详情 + 文件列表。

**请求 Query**：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | int | ✅ | `tasks.id` |

**响应 200**：
```json
{
  "id": 8,
  "status": "executed",
  "command_type": "mv",
  "commands": [
    "mv -i '/volume1/GV/a.mp4' '/volume1/GV/dst/a.mp4'",
    "mv -i '/volume1/GV/b.mp4' '/volume1/GV/dst/b.mp4'"
  ],
  "file_count": 2,
  "files": [
    {
      "file_id": 1,
      "file_path": "/volume1/GV/a.mp4",
      "file_name": "a.mp4",
      "md5": "178e097c...",
      "target_dir": "/volume1/GV/dst"
    }
  ],
  "created_at": "2026-07-22 07:30:00",
  "executed_at": "2026-07-22 07:31:00"
```

**注意**：
- `LEFT JOIN files` — 即使文件已被删除，关联行仍返回（`file_path` 来自 `files` 表快照）
- 不会因确认执行后文件删除而 404
- `commands` 是 JSON 数组（与 `/api/generate-*` 返回的 `command` 字符串不同）

**错误**：
| 状态码 | 消息 |
|---|---|
| 400 | `{"error": "缺少 id"}` |
| 404 | `{"error": "task 不存在"}` |

## 11. 公共字段速查

| 字段 | 类型 | 出现于 | 含义 |
|---|---|---|---|
| `readonly` | bool | 大部分查询/确认 API | config.json readonly 模式 |
| `library_locked` | bool | `/api/duplicates` `/api/tree` `/api/files` `/api/confirm-rm` `/api/confirm-mv` | 库内任一 MD5 出现 > 1 次 |
| `locked_group_count` | int | 同上 | 处于锁定的 MD5 组数 |
| `locked_md5_set` | string[] | `/api/tree` `/api/files` | 处于锁定的完整 MD5 列表 |
| `pending_task_exists` | bool | 所有查询 API | 是否有 pending task |
| `pending_task_count` | int | 同上 | pending task 数量 |
| `is_locked` | bool | `/api/duplicates` `/api/files` 单文件 | 该文件的 MD5 处于锁定 |
| `duplicate_ids` | int[] | `/api/import` | 本次导入后处于重复组的 file.id |

## 12. 锁定矩阵

| API | readonly 检查 | pending task 检查 | 库级锁定检查 |
|---|---|---|---|
| `/api/duplicates` | — | — | — |
| `/api/same-name` | — | — | — |
| `/api/tree` | — | — | — |
| `/api/files` | — | — | — |
| `/api/import` | ❌ | ❌ | ❌（**不受任何锁定影响**）|
| `/api/ignore-same-name` | ✅ | ❌ | ❌ |
| `/api/generate-rm` | ❌ | ✅ | ✅（隐式：每组留 1）|
