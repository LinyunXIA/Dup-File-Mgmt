## 0. RESTful 设计原则

本系统所有 User Story 与 API 遵循以下规则：

| 维度 | 规则 |
|---|---|
| 资源命名 | **复数名词**，用 `/api/<resources>`；子资源用 `/api/<parent>/{id}/<child>` |
| 动作语义 | **HTTP 方法承载动作**，不用动词命名资源 |
| GET | 读取（幂等，安全） |
| POST | 创建资源 / 触发非幂等动作 |
| PUT | 完整更新（幂等） |
| PATCH | 部分更新 |
| DELETE | 删除资源 |
| 路径层级 | 主资源 → 子资源 → 动作（动作只能是 `POST`，且为子资源级） |
| 查询参数 | 用于过滤、搜索、分页、排序 |
| 状态码 | 200（成功）/ 201（创建）/ 204（无内容）/ 400（参数错）/ 401（未认证）/ 403（禁止）/ 404（不存在）/ 409（冲突）/ 422（语义错）/ 500（服务器错）|
| 错误响应 | `{"error": {"code": "STRING_CODE", "message": "可读消息"}}` |
| 成功响应 | 创建返回 `201 + Location` 头；列表返回 `{data, pagination}` |
| 命名风格 | 路径用 kebab-case（`same-name`），JSON 字段用 snake_case |
| 版本控制 | 暂不引入（单用户内网工具，避免过度设计） |

### 资源清单

| 资源 | 路径 | 说明 |
|---|---|---|
| Files | `/api/files` | 文件元数据集合 |
| Directories | `/api/directories` | 目录树 |
| Duplicates | `/api/duplicates` | MD5 重复组（衍生资源，不在主表）|
| Same-name groups | `/api/same-name-groups` | 同名组（衍生资源）|
| Tasks | `/api/tasks` | 命令任务 |
| Config | `/api/config` | 系统配置（单例）|

## 2. 文件元数据（File Metadata）

### US-001 导入 md5_list.txt

**作为** 拥有文件服务器的内部用户
**我想要** 上传服务器生成的 `md5_list.txt` 到本系统
**以便于** 在浏览器端查重、整理文件元数据

**业务规则**：
- 支持全量（first）和增量（incremental）两种模式
- 增量模式自动过滤库里已有的 MD5
- 自动跳过 `.DS_Store` 文件
- 解析每行 `<32位MD5>  <绝对路径>` 格式
- 完成后返回重复组 ID 列表

**RESTful API**：

| Method | Path | 作用 | 状态码 |
|---|---|---|---|
| `POST` | `/api/files:import` | 触发批量导入 | 201 / 400 / 413 |

**请求**：`multipart/form-data`

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `file` | form | File | ✅ | md5_list.txt（≤ 500MB） |
| `mode` | form | enum | ❌ 默认 `first` | `first` / `incremental` |

**响应 201 Created**：
```json
{
  "data": {
    "imported": 12345,
    "updated": 678,
    "skipped_ds_store": 23,
    "filtered_by_existing_md5": 0,
    "errors": 0,
    "total_lines": 13046,
    "elapsed_ms": 1234,
    "duplicate_ids": [1, 5, 12]
  }
}
```

**错误**：
- 400 INVALID_FILE — 缺少 file 字段
- 400 INVALID_MODE — mode 值非法
- 413 FILE_TOO_LARGE — 超过 500MB
- 422 PARSE_ERROR — 大面积解析失败

## 3. 查重浏览（Duplicate Detection）

### US-004 查询 MD5 重复组

**作为** 用户
**我想要** 查看库内所有 MD5 重复的文件组
**以便于** 决定哪些文件可以删除

**业务规则**：
- 只列出出现次数 > 1 的 MD5 组
- 组内按 file_name 排序
- 每行带 `is_locked` 标记（前端决定 disabled 状态）
- 任意 MD5 重复组存在 → 库级锁定（所有写操作被拒）

**RESTful API**：

| Method | Path | 作用 | 状态码 |
|---|---|---|---|
| `GET` | `/api/duplicates` | MD5 重复组列表 | 200 |

**请求 Query**：

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `sort` | enum | `md5` | `md5` / `count` / `total_size` |
| `order` | enum | `asc` | `asc` / `desc` |

**响应 200 OK**：
```json
{
  "data": [
    {

      "md5_short": "178e097c5039",
      "count": 2,

        {
          "id": 1,

          "md5": "178e097c50395b96bdf286f1d8f88a44",
```

### US-006 忽略同名组

**作为** 用户
**我想要** 一次性忽略某个同名组
**以便于** 不再被它打扰

**业务规则**：
- 必须全选组内所有未忽略文件
- 写入 `files.ignored_at = NOW`

**RESTful API**：

| Method | Path | 作用 | 状态码 |
|---|---|---|---|
| `POST` | `/api/same-name-groups/{group_key}:ignore` | 标记同名组为已忽略 | 200 / 400 / 403 / 404 |

> `group_key` = URL 编码的 `file_name`

**请求 Body**：
```json
{ "file_ids": [10, 11, 12] }
```

**响应 200 OK**：
```json
{
  "data": {
    "ignored": 3,
    "group_key": "1.mp4"
  }
}
```

**错误**：
- 400 FILE_IDS_EMPTY — file_ids 为空
- 400 GROUP_NOT_FOUND — file_name 不存在或不是同名组
- 400 NOT_ALL_SELECTED — 未全选组内文件
- 403 READONLY_MODE — 系统只读

## 4. 文件操作（File Operations）

### US-007 批量删除文件

**作为** 用户
**我想要** 选中若干文件 → 生成 rm 命令 → 服务器执行
**以便于** 清理重复 / 不需要的文件

**业务规则**：
- 创建 pending task（不直接删）
- 路径用 `shlex.quote` 转义
- **1 文件 1 行** rm 命令
- 同 MD5 必须至少留 1 个
- Task 锁定期间拒绝创建新 task
- 库级锁定期间（MD5 重复未消完）通过 generate-rm 校验拦截

**RESTful API**：

| Method | Path | 作用 | 状态码 |
|---|---|---|---|
| `POST` | `/api/tasks` | 创建删除/移动任务 | 201 / 400 / 403 / 404 |
| `GET` | `/api/tasks/{id}` | 查询任务详情 | 200 / 404 |
| `POST` | `/api/tasks/{id}/execute` | 确认已执行 | 200 / 400 / 404 |
| `POST` | `/api/tasks/{id}/cancel` | 取消 pending 任务 | 200 / 400 / 404 |

> 删除和移动共用 `POST /api/tasks`，通过 `action` 字段区分。

**US-007 创建删除任务**：

**请求 Body**：
```json
{
  "action": "rm",
  "file_ids": [1, 5, 12]
}
```

**响应 201 Created** + `Location: /api/tasks/7`：
```json
{
  "data": {
    "id": 7,
    "status": "pending",
    "action": "rm",
    "command": "rm '/volume1/GV/a.mp4'\nrm '/volume1/GV/b.mp4'",
    "file_count": 2,
    "paths": ["/volume1/GV/a.mp4", "/volume1/GV/b.mp4"]
  }
}
```

**错误**：
- 400 FILE_IDS_EMPTY — file_ids 为空
- 400 ALL_SELECTED — 同 MD5 全选（必须留 1）
- 404 FILE_NOT_FOUND — 文件不存在
- 403 PENDING_TASK_EXISTS — task 全局锁

### US-008 批量移动文件

**作为** 用户
**我想要** 选中文件 + 选择目标目录 → 生成 mv 命令
**以便于** 整理文件结构

**业务规则**：
- 目标目录必须在库中存在
- `mv -i`（交互确认防覆盖）
- 多条用 ` && \` 串联
- 锁定期间拒绝（库级 + task 级）

**RESTful API**：复用 `POST /api/tasks`

**请求 Body**：
```json
{
  "action": "mv",
  "file_ids": [1, 5, 12],
  "target_dir": "/volume1/GV/dst"
}
```

**响应 201 Created** + `Location: /api/tasks/8`：
```json
{
  "data": {
    "id": 8,
    "status": "pending",
    "action": "mv",
    "command": "mv -i '/volume1/GV/a.mp4' '/volume1/GV/dst/a.mp4' && \\\nmv -i '/volume1/GV/b.mp4' '/volume1/GV/dst/b.mp4'",
    "file_count": 2,
    "new_paths": ["/volume1/GV/dst/a.mp4", "/volume1/GV/dst/b.mp4"]
  }
}
```

**错误**：
- 400 FILE_IDS_EMPTY / TARGET_DIR_EMPTY
- 400 TARGET_DIR_NOT_IN_LIBRARY — 目标目录在库中不存在
- 403 LIBRARY_LOCKED — 文件处于 MD5 重复锁定
- 403 PENDING_TASK_EXISTS — task 全局锁
- 404 FILE_NOT_FOUND

### US-009 播放文件

**作为** 用户
**我想要** 点 ▶️ 用 macOS 默认播放器打开文件
**以便于** 快速预览内容

**业务规则**：
- 拼 `smb://<ip>/<相对路径>`，剥路径第一级
- 协议白名单（只允许 smb://）
- 不受任何锁定影响

**RESTful API**：

| Method | Path | 作用 | 状态码 |
|---|---|---|---|
| `POST` | `/api/files/{id}/play` | 触发播放 | 200 / 400 / 404 / 500 / 503 |

**请求 Body**：空

**响应 200 OK**：
```json
{
  "data": {
    "smb_url": "smb://192.168.88.212/GV/a.mp4"
  }
}
```

**错误**：
- 400 FILE_ID_MISSING — 缺少 id
- 404 FILE_NOT_FOUND — 文件不存在
- 400 INVALID_PATH — 路径格式不对
- 503 SMB_NOT_CONFIGURED — config.json 未配 smb_server_ip
- 500 OPEN_FAILED — subprocess 异常

## 8. RESTful 设计与现状差异

> 本文档按 RESTful 理想规范设计；与 `app.py` 现状有差异，列出供后续重构参考。

| # | 现状 API | RESTful 化 | 差异说明 |
|---|---|---|---|
| 1 | `GET /api/tree` | `GET /api/directories` | 路径名调整 |
| 2 | `GET /api/same-name` | `GET /api/same-name-groups` | 复数化 |
| 3 | `POST /api/ignore-same-name` | `POST /api/same-name-groups/{key}:ignore` | 路径 + 动作子资源化 |
| 4 | `POST /api/generate-rm` | `POST /api/tasks` (action=rm) | 动词路径 → 资源 + 字段 |
| 5 | `POST /api/confirm-rm` | `POST /api/tasks/{id}/execute` | 动词路径 → 子资源动作 |
| 6 | `POST /api/generate-mv` | `POST /api/tasks` (action=mv) | 同 #4 |
| 7 | `POST /api/confirm-mv` | `POST /api/tasks/{id}/execute` | 同 #5 |
| 8 | `POST /api/play` | `POST /api/files/{id}/play` | 动词路径 → 子资源动作 |
| 9 | `POST /api/import` | `POST /api/files:import` | 动词路径 → 资源动作 |
| 10 | `GET /api/task-detail` | `GET /api/tasks/{id}` | 去 `-detail` 冗余 |
| 11 | `POST /api/cancel-task` | `POST /api/tasks/{id}/cancel` | 动词路径 → 子资源动作 |
| 12 | `GET /api/config-status` | `GET /api/config` | 单数化 + 去 `-status` |
| 13 | 错误响应 `{"error": "..."}` | `{"error": {"code": "...", "message": "..."}}` | 错误结构化 |
| 14 | 查询响应直接 `{}` 顶层 | `{data, pagination, meta}` | 分页 + 元数据标准封装 |
| 15 | 路径 `/api/files` 无分页 | 加 `?page=&page_size=` | 引入分页 |
| 16 | 增量导入无过滤 | 真正实现 `mode=incremental` 的 MD5 过滤 | 业务功能未实现 |

### 8.1 命名约定参考

| 旧（命令式） | 新（资源式） |
|---|---|
| `generate-rm` | `POST /api/tasks` (action=rm) |
| `generate-mv` | `POST /api/tasks` (action=mv) |
| `confirm-rm` / `confirm-mv` | `POST /api/tasks/{id}/execute` |
| `cancel-task` | `POST /api/tasks/{id}/cancel` |
| `ignore-same-name` | `POST /api/same-name-groups/{key}:ignore` |
| `play` | `POST /api/files/{id}/play` |
| `import` | `POST /api/files:import` |

> 注：Google API Design Guide 建议用 `POST /api/resources:action` 表示集合级动作；AWS 风格用 `POST /api/resources/action`。本设计采用 Google 风格（冒号）。

### 8.2 错误响应升级

**现状**：
```json
{ "error": "缺少 task_id" }
```

**RESTful 化**：
```json
{
  "error": {
    "code": "TASK_ID_REQUIRED",
    "message": "缺少 task_id"
  }
}
```

错误码采用 `SCREAMING_SNAKE_CASE`，便于前端 i18n 映射。

### 8.3 元数据封装

**现状**：每个查询 API 各自暴露 `library_locked` / `readonly` / `pending_task_*`

**RESTful 化**：统一封装到 `meta` 字段：

```json
{
  "data": [...],
  "pagination": {...},
  "meta": {
    "readonly": false,
    "library_locked": true,
    "locked_group_count": 343,
    "pending_task_exists": false,
    "pending_task_count": 0
  }
}
```

## 9. 状态码速查

| 状态码 | 场景 |
|---|---|
| 200 OK | 成功（读 / 动作已生效） |
| 201 Created | 资源已创建（POST /api/tasks、/api/files:import） |
| 204 No Content | （预留）DELETE 成功 |
| 400 Bad Request | 参数错误（缺字段、格式错、值非法） |
| 401 Unauthorized | （预留）鉴权失败 |
| 403 Forbidden | 权限不足（readonly、库级锁、task 锁） |
| 404 Not Found | 资源不存在 |
| 409 Conflict | （预留）资源状态冲突 |
| 413 Payload Too Large | 上传文件 > 500MB |
| 422 Unprocessable Entity | 语义错误（parse 失败、规则不满足） |
| 500 Internal Server Error | 服务器异常 |
| 503 Service Unavailable | 服务未就绪（SMB 未配置） |
